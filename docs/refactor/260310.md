# 260310 리팩터 기록 (monthlyStats 도입 및 Firestore read 최적화)

## 1) 배경

- Firestore Spark 플랜에서 `Document reads` 일일 무료 한도(5만)에 근접하는 상황이 발생.
- 기존 월간 캘린더 렌더링이 원본 컬렉션 4개 월 조회를 기반으로 동작하여, 사용자 데이터가 누적될수록 read가 증가하는 구조였음.

## 2) 이번 작업 목표

1. 월간 캘린더의 Firestore read 부담 완화
2. 기존 UX(토글 즉시 반영) 유지
3. 위험을 줄이기 위해 fallback 가능한 점진 도입

## 3) 주요 코드 변경 사항

### 3.1 monthlyStats 도메인 신규 추가

신규 파일:

- `src/shared/api/monthlyStats/types.ts`
- `src/shared/api/monthlyStats/refs.ts`
- `src/shared/api/monthlyStats/queries.ts`
- `src/shared/api/monthlyStats/index.ts`

핵심 기능:

- `getMonthlyStatsByMonthOnce`:
  - `users/{uid}/monthlyStats/{YYYY-MM}` 문서를 단건 조회
- `upsertMonthlyStatsByMonth`:
  - fallback 계산 결과를 월 문서로 저장
- `patchMonthlyStatsCompletionByDay`:
  - `days.{DD}`만 부분 갱신(merge)

### 3.2 React Query 키 추가

수정 파일:

- `src/shared/query/keys.ts`

추가:

- `monthlyStatsKeys.byMonth(userId, month)`

### 3.3 월간 캘린더 조회 경로 전환

수정 파일:

- `src/shared/hooks/calendar.ts`
- `src/pages/HomePage/components/MonthlyReport/MonthlyReportSection.tsx`

변경 내용:

- `useMonthlyData`가 `monthlyStats`를 우선 조회
- `monthlyStats` 문서가 없을 때만 기존 4쿼리 fallback 수행
  - tasks byMonth
  - taskLogs byMonth
  - routines byMonth
  - routineLogs byMonth
- fallback 성공 시 계산 결과를 `monthlyStats`로 upsert
- `useMonthlyActivityCountMap`가 `monthlyStats.days`를 우선 사용

### 3.4 루틴 토글 시 monthlyStats 동기화

수정 파일:

- `src/pages/HomePage/components/RoutineReport/RoutineReportSection.tsx`

변경 내용:

- 루틴 토글 시 `routineReport`/`routineLogs` 캐시 갱신 외에
- `monthlyStats` 캐시도 optimistic update
- 이후 `patchMonthlyStatsCompletionByDay`로 Firestore 문서 동기화

### 3.5 태스크 토글 404 이슈 수정 + monthlyStats 동기화

수정 파일:

- `src/shared/api/taskLog/crud.ts`
- `src/pages/HomePage/components/TaskReport/hooks/useTaskList.ts`

#### 3.5.1 404 이슈 원인/수정

원인:

- 빠른 토글 상황에서 임시 로그 ID(`temp-*`)를 `updateDoc` 대상으로 사용하면서 문서 미존재 404 발생

수정:

- canonical ID(`{taskId}_{date}`) 문서 기준으로 `setDoc` 저장
- `updateDoc` 실패 시 `setDoc` fallback

#### 3.5.2 태스크 토글 monthlyStats 반영

- 태스크 토글 시 `monthlyStats` 캐시 optimistic update
- 이후 `patchMonthlyStatsCompletionByDay` 호출로 문서 동기화
